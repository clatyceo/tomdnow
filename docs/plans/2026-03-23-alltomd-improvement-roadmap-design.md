# AllToMD 전체 개선 로드맵 설계

## 개요

AllToMD(tomdnow.com)의 경쟁 분석 결과를 바탕으로 한 전체 개선 로드맵.
접근 방식 C(병렬 레이어): SEO + 신뢰 동시 → 기능 확장 → 수익화 → 브랜드 구축.

## 전제 조건

- **실행 주체:** 1인 개발 + Claude 지원
- **인프라 예산:** 월 $20-50, 성장에 따라 유연하게 확장
- **최우선 과제:** 트래픽 확보 (#1) + 신뢰 구축 (#2)
- **수익 모델:** Freemium + Google AdSense
- **시장 전략:** 글로벌 동시 (8개 언어 SEO/GEO 선행 → GA4 데이터 기반으로 트래픽 상위 국가 집중 마케팅)

## Phase 1: SEO + 신뢰 구축 (1-4주)

### 1-A. SEO 강화

| 항목 | 내용 | 난이도 |
|------|------|--------|
| 블로그 시스템 | Next.js 내 MDX 기반 정적 블로그 (`/[locale]/blog/`) | 중 |
| 초기 콘텐츠 | 8개 언어 x 5-10편 (PDF→MD 변환법, 마크다운 활용법, 개발자 워크플로우 등) | 중 |
| Schema.org | SoftwareApplication + HowTo + FAQ 구조화 데이터 보강 | 낮 |
| 언어별 SEO 최적화 | 기존 `/[locale]/` 구조 활용, 도구별 개별 SEO 메타 최적화 | 낮 |
| 개발자 백링크 | DEV.to, Reddit, Hacker News 글 작성 | 낮 |
| Google Search Console | 사이트맵 제출, 인덱싱 모니터링 | 낮 |

### 1-B. 보안 신뢰 구축

| 항목 | 내용 | 난이도 |
|------|------|--------|
| 보안 전용 페이지 | `/[locale]/security` — 메모리 처리, 즉시 삭제, TLS, 서버 미저장 설명 | 낮 |
| 홈페이지 신뢰 배지 | "파일 서버 미저장", "TLS 암호화", "즉시 삭제" 아이콘 + 텍스트 | 낮 |
| 개인정보처리방침 | `/[locale]/privacy` 페이지 (GDPR 기본 준수) | 낮 |
| 이용약관 | `/[locale]/terms` 페이지 | 낮 |

### 1-C. 마케팅 기반

| 항목 | 내용 |
|------|------|
| Google Analytics 4 | 트래픽 소스, 국가별 유입 추적 |
| Google AdSense | 블로그 + 도구 페이지에 비침투적 광고 배치 |
| GEO 전략 | 8개 언어 SEO 선행 → GA4 데이터로 트래픽 상위 국가 식별 → 해당 국가 집중 마케팅 |

## Phase 2: 기능 확장 (1-3개월)

### 2-A. HWP/HWPX 지원 (최우선)

| 항목 | 내용 |
|------|------|
| 우선순위 | Phase 2 첫 번째 작업 |
| HWPX | XML 기반 (ZIP 안에 XML), 파싱 비교적 수월 |
| HWP (구형) | 바이너리 포맷, `python-hwp` / `olefile` 등 오픈소스 조사 → 부족 시 자체 파서 개발 |
| SEO 시너지 | "HWP to Markdown" 키워드 경쟁자 거의 없음 → 블로그 콘텐츠와 시너지 |

### 2-B. 배치 처리

| 항목 | 내용 |
|------|------|
| 프론트엔드 | 다중 파일 드래그앤드롭, 진행률 표시 |
| 백엔드 | `POST /api/convert/batch` — 병렬 처리 (asyncio), ZIP 다운로드 |
| 제한 | 무료: 5파일/회, 유료: 50파일/회 |

### 2-C. REST API

| 항목 | 내용 |
|------|------|
| 엔드포인트 | `POST /api/v1/convert` — 기존 백엔드 로직 재사용 |
| 인증 | API 키 기반 (무료 티어: 일 50회, 유료: 무제한) |
| 문서 | `/[locale]/docs/api` — OpenAPI/Swagger 자동 생성 + 사용 예시 |
| SDK | 추후 npm/pip 패키지 (Phase 4 이후) |

### 2-D. AI 기능

| 항목 | 내용 | 비용 전략 |
|------|------|-----------|
| OCR | 이미지/스캔 PDF → 텍스트 추출 → 마크다운 | Tesseract (무료 오픈소스) 우선, 품질 부족 시 Google Vision API |
| 문서 요약 | 변환 결과를 AI로 요약 | Claude API / OpenAI API (유료 기능으로만 제공해 비용 전가) |
| 스마트 포맷팅 | 테이블 자동 감지, 헤딩 구조화 | 자체 로직 (비용 없음) |

### 2-E. 추가 포맷

| 우선순위 | 포맷 |
|----------|------|
| 1순위 | HWP/HWPX (한국 시장 핵심) |
| 2순위 | RTF, TXT |
| 3순위 | Google Docs (URL 입력), Notion export |

## Phase 3: 수익화 (3-6개월)

### 3-A. Freemium 모델

| 티어 | 가격 | 포함 기능 |
|------|------|-----------|
| Free | $0 | 단일 파일 변환, 일 20회, 광고 노출 |
| Pro | $5-9/월 | 배치 처리, API 일 500회, 광고 제거, OCR |
| Team | $15-25/월/인 | 무제한 API, AI 요약, 우선 지원, SLA |

### 3-B. Google AdSense

| 항목 | 내용 |
|------|------|
| 배치 위치 | 블로그 글 중간/하단, 변환 결과 하단 (변환 UX 방해 금지) |
| 원칙 | 도구 사용 흐름에는 광고 최소화, 콘텐츠 페이지 중심 |

### 3-C. 결제 인프라

| 항목 | 내용 |
|------|------|
| 결제 | Stripe (글로벌) + 토스페이먼츠 (한국, 선택) |
| 사용자 시스템 | 이메일 기반 가입, API 키 발급/관리 대시보드 |
| 비용 관리 | AI API 호출은 유료 티어에서만 → 비용 전가 구조 |

## Phase 4: 브랜드 & 사회적 증명 (6개월+)

### 4-A. 리뷰 플랫폼

| 항목 | 시기 |
|------|------|
| Product Hunt 런칭 | Phase 2 API 출시 시점 |
| G2 / Capterra 등록 | Pro 플랜 출시 후 유료 사용자 리뷰 확보 |
| GitHub Stars | API SDK 오픈소스화로 개발자 커뮤니티 참여 유도 |

### 4-B. 커뮤니티 & 콘텐츠 마케팅

| 항목 | 내용 |
|------|------|
| DEV.to / Hashnode | 기술 블로그 시리즈 |
| Reddit / HN | API 출시, Product Hunt 런칭 시 포스팅 |
| YouTube / 블로그 | 사용법 튜토리얼, 비교 영상 |

### 4-C. 앱 확장

| 항목 | 내용 |
|------|------|
| VS Code 확장 | 에디터에서 바로 파일→마크다운 변환 |
| CLI 도구 | npm/pip 패키지로 터미널에서 변환 |
| 모바일 대응 | PWA 우선 (네이티브 앱은 ROI 검토 후) |

## 전체 타임라인

```
Week 1-4:   [Phase 1] SEO + 신뢰 구축 (병렬)
Month 2:    [Phase 2-A] HWP/HWPX 지원
Month 2-3:  [Phase 2-B,C] 배치 처리 + REST API
Month 3-4:  [Phase 2-D] AI 기능 (OCR, 요약)
Month 4-6:  [Phase 3] Freemium + AdSense + 결제
Month 6+:   [Phase 4] 브랜드, 리뷰, 앱 확장
```

## 성공 지표

| Phase | KPI |
|-------|-----|
| Phase 1 | 월간 오가닉 트래픽 1,000+ 방문, 보안 페이지 이탈률 < 60% |
| Phase 2 | 일간 변환 500+건, API 가입 100+명 |
| Phase 3 | 월 수익 $100+, 유료 전환율 2-5% |
| Phase 4 | Product Hunt Top 5, G2 리뷰 10+개 |
