# i18n Design Document

> Date: 2026-03-23
> Status: Approved

## Overview

ToMarkdown에 next-intl 기반 다국어 지원 추가. 8개 언어, SEO 로컬라이징 포함.

| 한국어 | ko | Korea |
| 日本語 | ja | Japan |

| Español | es | Spain/LatAm |
| Português | pt | Brazil/Portugal |
| Deutsch | de | Germany/DACH |
| Français | fr | France/Africa |

## Architecture

### Routing
- Pattern: `/[locale]/` prefix (e.g., `/ko/pdf-to-markdown`)
- Default: `en` (no redirect — `/` serves English)
- Middleware: auto-detect browser locale → redirect

### File Structure
```
frontend/
├── messages/
│   ├── en.json, ko.json, ja.json, zh.json
│   ├── es.json, pt.json, de.json, fr.json
├── i18n/
│   ├── config.ts        # locales, defaultLocale
│   ├── request.ts       # next-intl server config
│   └── navigation.ts    # createNavigation
├── middleware.ts         # locale detection + redirect
├── app/
│   └── [locale]/        # all routes under locale
│       ├── layout.tsx
│       ├── page.tsx
│       ├── [15 tool pages]/page.tsx
│       ├── about/page.tsx
│       ├── privacy/page.tsx
│       └── terms/page.tsx
```

### Translation Key Structure
```json
{
  "common": { "convert", "download", "copy", "dragDrop", ... },
  "nav": { "documents", "data", "media", ... },
  "home": { "title", "subtitle", "features", ... },
  "tools": {
    "pdf": { "title", "h1", "displayName", "subtitle", "seo", "howTo", "whyConvert", "faq" },
    ...15 tools
  },
  "seo": { "howToTitle", "whyTitle", "faqTitle" },
  "about": { ... },
  "privacy": { ... },
  "terms": { ... }
}
```

### Translation Approach
- Claude agents with SEO-localization prompt
- Each language: search-pattern-aware keywords
- Not literal translation — native phrasing

### Date/Number Formatting
- `useFormatter()` from next-intl
- Date: locale-aware (e.g., "March 2026" / "2026년 3월")
- Number: locale-aware separators

### SEO
- hreflang tags on all pages
- Locale-specific metadata (title, description, keywords)
- Sitemap with locale-prefixed URLs
- Per-language tool keywords (not translations of English keywords)
